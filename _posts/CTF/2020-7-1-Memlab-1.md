MemLabs is an educational, introductory set of CTF-styled challenges which is aimed to encourage students,security researchers and CTF  players to get started with the field of **Memory Forensics**.

## **Challenge Description**
>
> My sister's computer crashed. We were very fortunate to recover this  memory dump. Your job is get all her important files from the system.  From what we remember, we suddenly saw a black window pop up with some  thing being executed. When the crash happened, she was trying to draw  something. Thats all we remember from the time of crash.
>
> **Note**: This challenge is composed of 3 flags.
>
> **Challenge file**: [MemLabs_Lab1](https://mega.nz/#!6l4BhKIb!l8ATZoliB_ULlvlkESwkPiXAETJEF7p91Gf9CWuQI70)

Let's identify the profile first , for that we can use `imageinfo` plugin.

```
$ volatility -f MemoryDump_Lab1.raw imageinfo
```

![](/assets/img/posts/memlabs/lab1/1.png)
To list the processes that run on  the  system when this dump was taken , let's use [**pslist**](https://github.com/volatilityfoundation/volatility/wiki/Command-Reference#pslist) plugin 
```
vol.py -f MemoryDump_Lab1.raw --profile = Win7SP1x64 pslist
```
from the description , the most supsious processes are these : 
![](/assets/img/posts/memlabs/lab1/2.png)

To finds commands that attackers typed into cmd.exe , we can use [**consoles**](https://github.com/volatilityfoundation/volatility/wiki/Command-Reference#consoles) plugin
```
vol.py -f MemoryDump_Lab1.raw --profile=Win7SP1x64 consoles 
```
here we can recognize a **base64** string 
![](/assets/img/posts/memlabs/lab1/3.png)

after decode it , we get our first flag . 
```
echo ZmxhZ3t0aDFzXzFzX3RoM18xc3Rfc3Q0ZzMhIX0= | base64 -d 
```
>flag{th1s_1s_th3_1st_st4g3!!}

Now , let's continue with  `mspaint.exe` process , from description he told us : "his sister was trying to draw something" and I knew from [here](https://w00tsec.blogspot.com/2015/02/extracting-raw-pictures-from-memory.html) that we can extract the RAW picture from the dump . 
first , let's use [memdump](https://github.com/volatilityfoundation/volatility/wiki/Command-Reference#memdump) plugin 
```
vol.py -f MemoryDump_Lab1.raw --profile=Win7SP1x64 memdump -p 2424 -D output/
```
Then , let's use **gimp** but first we need to rename the file from **.dmp to .data** 
so gimp can open the file  . 
after playing with width and length then flip the image we found our flag . 
![](/assets/img/posts/memlabs/lab1/6.png)

>flag{G00d_BoY_good_girL}


Now , let's see what `WinRAR.exe` have 
,I used [**filescan**](https://github.com/volatilityfoundation/volatility/wiki/Command-Reference#filescan) plugin and filter the output with **grep** to file **.rar** files only 
```
vol.py -f MemoryDump_Lab1.raw --profile=Win7SP1x64 filescan | grep .rar
```
![](/assets/img/posts/memlabs/lab1/4.png)
I found file called **Important.rar** , let's dump it with [**dumpfiles**](https://github.com/volatilityfoundation/volatility/wiki/Command-Reference#dumpfiles) plugin and see what its content . 
```
vol.py --profile Win7SP1x64 dumpfiles -Q 0x000000003fa3ebc0 -D output/
```
I tried to unrar the file but it's password-protected , I see its **strings** and found this import one : `CMTPassword is NTLM hash(in uppercase) of Alissa's account passwd.` 

After searching for hashes plugins in volatility , I found [**hashdump**](https://github.com/volatilityfoundation/volatility/wiki/Command-Reference#hashdump)
```
vol.py -f MemoryDump_Lab1.raw --profile=Win7SP1x64 hashdump
```
![](/assets/img/posts/memlabs/lab1/5.png)



so , the password is **F4FF64C8BAAC57D22F22EDC681055BA6** (convereted to  uppercase )
and we get our third and last flag .
