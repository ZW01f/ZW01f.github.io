> ## **Challenge Description**
>
>A malicious script encrypted a very secret piece of information I had on my system. Can you recover the information for me please?
>
> Note-1: This challenge is composed of only 1 flag. The flag split into 2 parts.
Note-2: You'll need the first half of the flag to get the second.
You will need this additional tool to solve the challenge,
>
```
  sudo apt install steghide
```
>The flag format for this lab is: inctf{s0me_l33t_Str1ng}

> **Challenge file**: [MemLabs_Lab3](https://mega.nz/#!2ohlTAzL!1T5iGzhUWdn88zS1yrDJA06yUouZxC-VstzXFSRuzVg)

OK , Let's identify the profile first , for that we can use `imageinfo` plugin.


```
$ volatility -f MemoryDump_Lab3.raw imageinfo
```
![1.png](/assets/images/memlabs/lab3/1.png)
To list the processes that run on  the  system when this dump was taken , let's use [**pslist**](https://github.com/volatilityfoundation/volatility/wiki/Command-Reference#pslist) plugin 
```
$ vol.py -f MemoryDump_lab3.raw --profile = Win7SP1x64 pslist
```



![2.png](/assets/images/memlabs/lab3/2.png)

 

first , I used **iehistory** plugin to see  internet explorer history . 

```
$ vol.py  -f MemoryDump_Lab3.raw --profile=Win7SP1x86 iehistory
```
![3.png](/assets/images/memlabs/lab3/3.png)

we see some important files here, let's begin with the python script 


```
$ vol.py  -f MemoryDump_Lab3.raw --profile=Win7SP1x86 filescan > scanfiles.txt
```

```
$ vol.py  -f MemoryDump_Lab3.raw --profile=Win7SP1x86 dumpfiles -Q 0x000000003de1b5f0 -D output/
```


![4.png](/assets/images/memlabs/lab3/4.png)

now let's dump and see what **vip.txt** have . 
```
$ vol.py  -f MemoryDump_Lab3.raw --profile=Win7SP1x86 dumpfiles -Q 0x000000003e727e50 -D output/
```
with **cyberchef** we have the first part of the flag : 


![5.png](/assets/images/memlabs/lab3/5.png)


As the description  said we will  use  **steghide** which is a jpg steganography tool, so let's dump **suspision1.jpeg** file 
```
$ vol.py  -f MemoryDump_Lab3.raw --profile=Win7SP1x86 dumpfiles -Q 0x0000000004f34148 -D output/
```
to extract the secret we need a passphrase ,  The description said **You'll need the first half of the flag to get the second** so I tried the first half of the flag as the passphrase.


![6.png](/assets/images/memlabs/lab3/6.png)

finaly, we have our flag :) 
> inctf{0n3_h4lf_1s_n0t_3n0ugh}
