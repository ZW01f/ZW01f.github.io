> ## **Challenge Description**
>
> One of the clients of our company, lost the access to his system due  to an unknown error. He is supposedly a very popular "environmental"  activist. As a part of the investigation, he told us that his go to  applications are browsers, his password managers etc. We hope that you  can dig into this memory dump and find his important stuff and give it  back to us.
>
> **Note**: This challenge is composed of 3 flags.
>
> **Challenge file**: [MemLabs_Lab2](https://mega.nz/#!ChoDHaja!1XvuQd49c7-7kgJvPXIEAst-NXi8L3ggwienE1uoZTk)

First we need to identify the profile .
```
$ vol.py -f MemoryDump_Lab2.raw imageinfo
```
![1.png](/assets/images/memlabs/lab2/1.png)
from the description , you can note the  word "environmental". I decided to use `envars`To display a process's environment variables first .

```
$vol.py -f MemoryDump_Lab2.raw --profile=Win7SP1x64 envars
```
![2.png](/assets/images/memlabs/lab2/2.png)

so we found that base64 

```
echo ZmxhZ3t3M2xjMG0zX1QwXyRUNGczXyFfT2ZfTDRCXzJ9 |base64 -d 
```
and found a flag . 

>flag{w3lc0m3_T0_$T4g3_!_Of_L4B_2} 

Next, let’s check the processes list

![pslist.png](/assets/images/memlabs/lab2/pslist.png)

First interesting process is **KeePass.exe** : 
>KeePass Password Safe is a free and open-source password manager primarily for Windows and stores the passwords in a database with the extension **.kdbx** and looks it with a master password. 

```
$vol.py -f MemoryDump_Lab2.raw --profile=Win7SP1x64 filescan | grep ".kdbx"
```


![dump1.png](/assets/images/memlabs/lab2/dump1.png)


Let's dump it . 
```
$vol.py -f MemoryDump_Lab2.raw --profile=Win7SP1x64 dumpfiles -Q 0x000000003fb112a0 -D output/
```
of course , we need a master password to open it so let's search for passwords in files . 


```
$vol.py -f MemoryDump_Lab2.raw --profile=Win7SP1x64 filescan | grep -i  "password"

```

![passord_img](/assets/images/memlabs/lab2/passord_img.png)

let's dump it to see what does it have . 
```
$vol.py -f MemoryDump_Lab2.raw --profile=Win7SP1x64 dumpfiles -Q 0x000000003fce1c70 -D output/
```


![password.png](/assets/images/memlabs/lab2/password.png)

Now let’s use this  to open the database in KeePass.


![password.png](/assets/images/memlabs/lab2/keepass.png)

just copy this password , you'll find that is our second flag .
